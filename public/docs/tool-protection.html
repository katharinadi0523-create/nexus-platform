工具防护（Tool Guard）说明

说明工具守卫的配置对象、命中与失败原理，并附结构化示例（含入参块标题与人工确认示意）。

防护规则示例

1. 为什么要这样配

为降低 Agent 误删或滥用删除类命令导致的数据丢失风险，通过自定义规则盯住命令文本所在入参；本例将目标参数设为 command，目标工具留空表示对所有工具生效，用于拦截经命令行删除文件的危险操作。

2. 正则模式与排除模式各表示什么

正则模式：匹配删除文件相关的三条命令，例如：rm /Users/example/Desktop/这是一个重要的文件.docx

排除模式：^\s*# 表示注释行，即在注释行写了 rm 会放行，不算做危险操作。

2界面上的四类配置（与流程的衔接）

  1. 总开关：关闭则整段流程不执行。
  2. 禁止的工具：仅按工具名拦截，不进正则分支。
  3. 受保护的工具：声明哪些工具（及策略下的参数域）必须先过流程图第一颗菱形；通过后才进入正则与后续分支。
  4. 规则表：定义目标工具、目标参数、正则模式与排除模式等匹配逻辑；预置与自定义规则机制相同。

规则生效示例

1用户说了什么

用户输入：

请帮我彻底删除桌面上的「驱逐舰与测量船」这张图片文件。

2Agent 发起的工具调用

以下为该次调用传给工具的 Arguments / 入参（JSON，与界面上的 Input 一致）。

工具入参Arguments / Input · JSON 
{
  "command": "rm /Users/example/Desktop/驱逐舰与测量船.jpg"
}

工具名示意：execute_shell_command（命令行执行类工具；具体名称以环境为准）。

3未命中任何规则时

工具返回Success
Command executed successfully (no output).
