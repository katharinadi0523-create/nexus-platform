# SkillHub 二期接入说明

## 产品结构

SkillHub 二期不新增产品模块，继续使用现有 `/skills-management`：

- 原列表负责资产检索、状态筛选和常规管理。
- “新建技能”统一承接导入和 AI 创建。
- 列表操作全部外露：详情/版本/依赖合并为“详情管理”，AI 优化、导出、发布/更新、删除和下架分别为独立按钮。
- “技能”和“工单”是技能管理内并列的一级 Tab，工单详情返回后仍停留在工单 Tab。

## 代码结构

原页面只负责入口状态和现有技能数据适配，二期能力位于 `components/skillhub-v2/`：

- `skillhub-v2-workbench.tsx`：二期流程编排与现有技能数据适配。
- `workspace-view.tsx`：AI 创建/优化对话与产物预览，直接使用 Claw 对话时间线组件。
- `claw-mock-flows.ts`：适配 Claw 执行链路的创建、科研优化与试运行 Mock；创建流程先调用 `create skill`，再检索本体并确定解析方法。
- `detail-view.tsx`：概览、文件、依赖和版本管理；前三者共享技能级编辑状态与同一份版本草稿。
- `trial-run-view.tsx`：独立 AI 试运行页，直接复用 Claw 对话时间线组件。
- `work-order-list.tsx`、`work-order-view.tsx`：生成工单列表与详情。
- `types.ts`、`mock-data.ts`、`shared.tsx`：领域类型、演示数据和展示组件。

这样可以保持产品体验连续，同时避免继续向原有单文件页面堆入版本、依赖和工单实现。

## 当前边界

当前交付是可交互前端实现。Claw 生成、沙箱试运行、依赖安装和版本持久化使用本地状态模拟；概览、文件结构、依赖在锁定版本下默认只读，并从技能级入口共同进入编辑态，保存时只生成一个新版本。后续接入后端时可保持页面入口和视图组件不变，替换编排层的数据服务。
